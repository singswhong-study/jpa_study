1. 조회
    - find(클래스, id)
    - 조회 후 객체의 null 체크 필요

2. 수정
    - 트랜잭션 범위 내에서 변경된 값을 자동 반영
    - 필드값이 변경되면 commit 시점에 update
    - 비교는 영속컨텍스트에 있는 1차 캐시의 snapshot 과 비교한 후 결정

3. 삭제
    - find() 로 읽어온 객체를 em.remove() 로 제거
    - 어떤 트랜잭션에서 삭제 시도전에 다른 프로세스가 해당 데이터를 삭제한다면 에러 발생. => 익셉션 처리 중요

4. 삽입
    - 대상 객체를 persist() 해서 1차 캐시, 즉 영속성 컨텍스트에 넣어두고 commit 시점에 db 반영
